## 문제

지도가 주어지면 모든 지점에 대해서 목표지점까지의 거리를 구하여라.

문제를 쉽게 만들기 위해 오직 가로와 세로로만 움직일 수 있다고 하자.

## 입력

지도의 크기 n과 m이 주어진다. n은 세로의 크기, m은 가로의 크기다.(2 ≤ n ≤ 1000, 2 ≤ m ≤ 1000)

다음 n개의 줄에 m개의 숫자가 주어진다. 0은 갈 수 없는 땅이고 1은 갈 수 있는 땅, 2는 목표지점이다. 입력에서 2는 단 한개이다.

## 출력

각 지점에서 목표지점까지의 거리를 출력한다. 원래 갈 수 없는 땅인 위치는 0을 출력하고, 원래 갈 수 있는 땅인 부분 중에서 도달할 수 없는 위치는 -1을 출력한다.

# 회고

BFS 알고리즘을 이용해 풀이하였다.

목표지점에서 탐색하여 각 위치에서부터 목표지점까지의 거리를 구하면 됐다.

도달할 수 없는 위치는 -1을 출력해야하기 때문에 리스트를 하나 만들어 방문했는지의 여부를 체크해줬어야 했다.

처음 풀이는 출력할 때 -2씩 빼줘서 출력했는데 이는 운이 좋아서 맞은 풀이인것 같다. 만약 도달할 수 없는 위치의 값이나 갈 수 있는 땅의 위치의 값이 바뀌면 틀렸을 것이다.

for문을 통해 하나씩 탐색하면서 값이 2이면 BFS 탐색을 진행하는 식으로 코드를 작성하였는데, 이렇게 하면 BFS 탐색을 완료한 그래프에서 값이 2인 곳이 있으면 다시 BFS 탐색을 진행하기 때문에 탐색 결과가 이상하게 나왔다. 지도에서 값을 갱신해나가는 방식으로 코드를 짜서 그런 것 같다.

따라서 지도에서 값이 2인 곳을 찾으면 탐색을 완료한 후 그 다음에 더 이상 탐색을 하지 못하게 코드를 작성하거나, 새로운 빈 그래프에 정답을 채워 넣는 식으로 코드를 작성해야 했다.
